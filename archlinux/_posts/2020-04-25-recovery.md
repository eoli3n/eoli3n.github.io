``mkinitcpio`` defaulty generates a [fallback initramfs](https://wiki.archlinux.org/index.php/Mkinitcpio#Image_creation_and_activation) to recover if a kernel module fails.
Sadly, i had to recover sometimes because of a failing pacman upgrade.
If there is a kernel upgrade, [a pacman hook](https://git.archlinux.org/mkinitcpio.git/tree/libalpm/hooks/90-mkinitcpio-install.hook) automatically regenerates initramfs. But pacman hooks are executed after the upgrade is complete, sometimes pacman fails to upgrade a package and then the hook is forgotten... Just reboot and you're done.

Another case which require to live boot is if you need to do something on "/". Like [restoring a BTRFS slash snapshot](https://wiki.archlinux.org/index.php/Snapper#Restoring_/_to_a_previous_snapshot_of_@).

If you're host is your only one and you don't have any bootable usb stick, how to recover ?
A forgotten one in a drawer could work but what if it is too old ?

## What would be the perfect recover environnement ?

* A bootable Archlinux system
* Always up to date
* Easy to implement
* Not configuration dependant (disks, mounts...)
* Working when you need it

Since OSX's installer is dematerialized, there is just one key to press at boot to run the installer.
The latest installer is downloaded in RAM and booted. It leads to the good solution : netboot install.

In fact, it already exists, but it is [not well referenced](https://wiki.archlinux.org/index.php/Category:System_recovery) by the official wiki.

### How to netboot archlinux ?

2 docs exists : 
* [https://www.archlinux.org/releng/netboot/](Archlinux.org Netboot page)
* [https://wiki.archlinux.org/index.php/Netboot](Archlinux Netboot wiki)

The good news is that we just need to put 1 or 2 little files in the right place.

1. Install AUR [https://aur.archlinux.org/packages/ipxe-netboot](*ipxe-netboot*) 
   That package will generates the EFI/bios iPXE files.

2. Create a dir in your EFI dir

3. Hardlink */usr/share/ipxe-netboot/ipxe.efi* to */boot/EFI/netboot/*

4. Generates an entry to chainload that efi file. ([example for grub](https://wiki.archlinux.org/index.php/GRUB#gdisk))

You should now be able to netboot Archlinux Installer from your bootloader.
